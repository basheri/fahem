# Contributing

1. Read `CLAUDE.md` and the relevant specifications.
2. Create an issue or task tied to an acceptance criterion.
3. Implement one coherent vertical slice per pull request.
4. Add tests and update contracts/documentation with code changes.
5. Run the repository verification command before opening a pull request.
6. Never commit secrets, real student PDFs, or copyrighted test documents without explicit permission.
7. Use synthetic or properly licensed fixtures.

Pull requests must describe the problem, design, security/privacy impact, test evidence, screenshots for UI changes, migration impact, deletion-path impact, and rollback approach.
