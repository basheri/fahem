Read the relevant specification, acceptance criteria, traceability entries, current code, tests, and recent diff. Produce:

1. User outcome and explicit non-goals.
2. Components and files likely to change.
3. Data, API, security, privacy, deletion, and Arabic UX impacts.
4. Test-first plan.
5. Migration and rollback needs.
6. Ordered implementation tasks small enough to verify independently.
7. Definition of done with commands and observable evidence.
