# Documentation and Contract Rules

- Keep specifications aligned with implemented behavior.
- Use exact, testable language for requirements.
- Update OpenAPI for every endpoint or response change.
- Update JSON Schemas and generated types together.
- Record assumptions, limits, privacy behavior, and failure modes.
- Mermaid diagrams must match the actual components and data flows.
- Do not document a feature as complete without verification evidence.
